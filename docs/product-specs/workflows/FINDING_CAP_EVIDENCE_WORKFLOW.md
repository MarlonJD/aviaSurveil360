# Finding CAP Evidence Workflow

## Purpose

Manage finding follow-up from issue to closure.

## Steps

1. Create finding
2. Issue to auditee
3. Auditee submits CAP
4. CAA reviews CAP
5. CAP accepted or rejected
6. Auditee uploads evidence
7. CAA reviews evidence
8. Close finding or request more info

## Rules

- CAP accepted is not closure
- Rejected CAP/evidence needs auditee-visible reason
- Evidence versions are kept
- Closure requires authority

## UX notes

- Show current owner, due date and next action at the top of the screen.
- Keep history in timeline/tab, not as primary content.
- Use primary buttons that match the next action.
