# Acceptance Criteria and Feedback Questions

## Demo acceptance criteria

- Demo can be understood in under five minutes.
- Role switching is clear.
- Manager sees risk and overdue items.
- Inspector sees assigned work.
- Auditee sees only required actions.
- Checklist creates a finding.
- Finding detail shows lifecycle stepper.
- Auditee submits CAP.
- Auditee uploads evidence.
- Inspector accepts evidence.
- Finding closes only after evidence acceptance.
- Dashboard updates.
- Auditee cannot see internal notes.

## MVP acceptance criteria

- Organizations can be managed.
- Audits can be planned and assigned.
- Checklists can be executed.
- Findings can be issued.
- Auditees can respond through portal.
- CAP and evidence can be reviewed.
- Due-date reminders work.
- Overdue findings are visible.
- Manager dashboard shows key KPIs.
- Audit trail records critical actions.

## Stakeholder feedback questions

1. Does the workflow match CAA surveillance expectations?
2. Who should issue findings?
3. Who should close findings?
4. Is evidence always required before closure?
5. Should auditee request due-date extension?
6. Which audit domain should be demoed first?
7. Which dashboard metric matters most?
8. What should stay Phase 2?
