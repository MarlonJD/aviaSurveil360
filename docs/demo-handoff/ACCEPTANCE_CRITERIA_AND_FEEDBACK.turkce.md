# Acceptance Criteria ve Feedback Soruları

## Demo acceptance criteria

- Demo beş dakikadan kısa sürede anlaşılır.
- Role switching nettir.
- Manager risk ve overdue items görür.
- Inspector assigned work görür.
- Auditee sadece required actions görür.
- Checklist finding oluşturur.
- Finding detail lifecycle stepper gösterir.
- Auditee CAP submit eder.
- Auditee evidence upload eder.
- Inspector evidence accept eder.
- Finding sadece evidence acceptance sonrası kapanır.
- Dashboard güncellenir.
- Auditee internal notes göremez.

## MVP acceptance criteria

- Organizations yönetilebilir.
- Audits planlanıp atanabilir.
- Checklists yürütülebilir.
- Findings issue edilebilir.
- Auditees portal üzerinden cevap verebilir.
- CAP ve evidence review edilebilir.
- Due-date reminders çalışır.
- Overdue findings görünür.
- Manager dashboard key KPI'ları gösterir.
- Audit trail critical actions kaydeder.

## Stakeholder feedback soruları

1. Workflow CAA surveillance beklentisine uyuyor mu?
2. Finding'i kim issue etmeli?
3. Finding'i kim close etmeli?
4. Closure öncesi evidence her zaman gerekli mi?
5. Auditee due-date extension request yapabilmeli mi?
6. İlk demo hangi audit domain ile yapılmalı?
7. Hangi dashboard metriği en önemli?
8. Ne Phase 2'de kalmalı?
