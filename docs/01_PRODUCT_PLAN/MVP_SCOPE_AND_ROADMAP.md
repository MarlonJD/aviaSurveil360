# MVP Scope and Roadmap

## MVP goal

Prove the complete oversight loop with one strong scenario:

**Operator Audit → Checklist → Finding → Auditee CAP → Evidence Upload → CAA Review → Closure → Dashboard Update**

## MVP must include

1. Organization Registry
2. Audit Planning / Calendar
3. Audit Detail
4. Checklist Runner
5. Finding Management
6. Auditee Portal
7. CAP Submission
8. Evidence Upload
9. CAA Evidence Review
10. Due-Date Reminder Rules
11. Manager Dashboard
12. Basic Report Preview
13. Audit Trail
14. Admin Template Preview

## MVP must not include

- Offline mobile app
- Advanced risk-based planning engine
- AI finding classification
- Full regulation tree editor
- Complex workflow designer
- Full enforcement case management
- Advanced BI/report builder
- Deep integration with all AVIA products

## Roadmap

### Phase 0 — Demo-first prototype

Front-end-only clickable demo with mock data. Validate UX and story before backend.

### Phase 1 — Core MVP

Build web app for organization, audit, checklist, finding, CAP, evidence and dashboard.

### Phase 2 — Operational hardening

Email integration, file storage, role permissions, exports, template versioning, security and audit trail hardening.

### Phase 3 — Risk-based oversight

Organization risk profile, repeat finding analysis, risk-based plan suggestions, Oversight Health Index.

### Phase 4 — Mobile/offline and integrations

Mobile inspection app, offline checklist, photo capture, e-signature, AviaLicense/Enforcement integration.

## MVP success criteria

- A stakeholder understands the product in one demo.
- Inspector can see next action immediately.
- Auditee can submit CAP and evidence without e-mail.
- Manager can see critical/overdue findings.
- Finding closure is traceable.
