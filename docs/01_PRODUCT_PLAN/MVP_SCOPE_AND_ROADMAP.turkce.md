# MVP Kapsamı ve Roadmap

## MVP hedefi

Tek güçlü senaryo ile complete oversight loop'u kanıtlamak:

**Operator Audit → Checklist → Finding → Auditee CAP → Evidence Upload → CAA Review → Closure → Dashboard Update**

## MVP'de mutlaka olmalı

1. Organization Registry
2. Audit Planning / Calendar
3. Audit Detail
4. Checklist Runner
5. Finding Management
6. Auditee Portal
7. CAP Submission
8. Evidence Upload
9. CAA Evidence Review
10. Due-Date Reminder Rules
11. Manager Dashboard
12. Basic Report Preview
13. Audit Trail
14. Admin Template Preview

## MVP'de olmamalı

- Offline mobile app
- Advanced risk-based planning engine
- AI finding classification
- Full regulation tree editor
- Complex workflow designer
- Full enforcement case management
- Advanced BI/report builder
- Tüm AVIA ürünleriyle deep integration

## Roadmap

### Phase 0 — Demo-first prototype

Mock data ile front-end-only clickable demo. Backend öncesi UX ve hikaye doğrulanır.

### Phase 1 — Core MVP

Organization, audit, checklist, finding, CAP, evidence ve dashboard için web app.

### Phase 2 — Operational hardening

Email integration, file storage, role permissions, exports, template versioning, security ve audit trail hardening.

### Phase 3 — Risk-based oversight

Organization risk profile, repeat finding analysis, risk-based plan suggestions, Oversight Health Index.

### Phase 4 — Mobile/offline ve integrations

Mobile inspection app, offline checklist, photo capture, e-signature, AviaLicense/Enforcement integration.

## MVP başarı kriterleri

- Stakeholder ürünü tek demoda anlar.
- Inspector next action'ı hemen görür.
- Auditee e-mail olmadan CAP ve evidence submit eder.
- Manager critical/overdue finding'leri görür.
- Finding closure izlenebilirdir.
